Java 并发编程基础

发布于 2024年7月28日 | Java

Java 从诞生之初就内置了对多线程的支持，并发编程是 Java 语言中一个非常重要且核心的部分。理解 Java 并发编程，对于构建高性能、高吞吐量的服务端应用至关重要。

线程 (Thread)

线程是操作系统能够进行运算调度的最小单位。在 Java 中，我们可以通过两种主要方式创建线程：

  1. 继承 `Thread` 类。
  2. 实现 `Runnable` 接口。

推荐使用实现 `Runnable` 接口的方式，因为它避免了 Java 单继承的限制，并且能更好地实现代码与业务逻辑的分离。


class MyRunnable implements Runnable {
    public void run() {
        System.out.println("MyRunnable is running.");
    }
}

// In main method
Thread thread = new Thread(new MyRunnable());
thread.start();

锁和同步 (Locks and Synchronization)

当多个线程需要访问共享资源时，为了避免数据不一致和竞态条件，我们需要使用同步机制。Java 提供了多种同步机制：

  • `synchronized` 关键字：这是最基本的同步方式，可以修饰方法或代码块，确保同一时间只有一个线程可以执行被保护的代码。
  • `Lock` 接口：`java.util.concurrent.locks` 包提供了更灵活的锁机制，如 `ReentrantLock`。它提供了 `tryLock`、可中断的锁获取等 `synchronized` 不具备的功能。
  • `volatile` 关键字：`volatile` 是一种轻量级的同步机制，它能保证被修饰变量的"可见性"，即一个线程修改了变量的值，新值对其他线程是立即可见的。但它不保证原子性。

并发集合

`java.util.concurrent` 包提供了一系列线程安全的集合类，它们的性能通常优于使用 `Collections.synchronized...()` 方法包装的传统集合。常用的并发集合包括：

  • `ConcurrentHashMap`：线程安全的哈希表。
  • `CopyOnWriteArrayList`：适用于读多写少场景的线程安全列表。
  • `BlockingQueue`：阻塞队列，常用于生产者-消费者模式。

总结

Java 的并发世界博大精深，本文只是介绍了冰山一角。从基础的 `Thread` 和 `synchronized`，到高级的 `Executor` 框架、`Fork/Join` 框架和各种并发工具类，Java 为开发者提供了强大的武器库来应对复杂的并发挑战。
